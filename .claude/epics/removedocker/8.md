# Task: Add auto-setup scripts to root package.json

## Description
Add a `setup` script to the root `package.json` that handles first-run database initialization, and update the `dev` script to auto-run setup if no database exists yet.

## Acceptance Criteria
- [ ] `setup` script added: copies `.env.example` → `.env` (if missing), generates Prisma client, pushes schema, seeds database
- [ ] `dev` script updated: checks for `backend/prisma/dev.db`, runs setup if missing, then starts servers
- [ ] `db:generate` script added to root (delegates to backend workspace)
- [ ] Running `npm run dev` on a fresh clone auto-creates the database and seeds it
- [ ] Running `npm run dev` on subsequent runs skips setup and starts normally

## Technical Details

### Root `package.json` scripts:
```json
{
  "setup": "test -f .env || cp .env.example .env && npm run db:generate && npm run db:push && npm run db:seed",
  "dev": "test -f backend/prisma/dev.db && concurrently \"npm run dev -w backend\" \"npm run dev -w frontend\" || npm run setup && concurrently \"npm run dev -w backend\" \"npm run dev -w frontend\"",
  "db:generate": "npm run db:generate -w backend"
}
```

### Backend `package.json`:
- Ensure `db:generate` script exists: `prisma generate`

## Dependencies
- [ ] Task 001 (schema must be SQLite) for `db:push` to create a `.db` file
