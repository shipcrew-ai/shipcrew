# Task: Switch Prisma schema to SQLite with JSON array fields

## Description
Change the Prisma datasource provider from `postgresql` to `sqlite` and convert the two `String[]` fields on the Agent model to `String` with JSON defaults. SQLite does not support array column types, so these must store JSON-encoded arrays as plain strings.

## Acceptance Criteria
- [ ] `datasource db` provider changed from `"postgresql"` to `"sqlite"`
- [ ] `datasource db` url changed to `env("DATABASE_URL")` (no change needed if already using env)
- [ ] `Agent.skills` changed from `String[]` to `String @default("[]")`
- [ ] `Agent.channels` changed from `String[]` to `String @default("[]")`
- [ ] No other models require changes (verify all other fields are SQLite-compatible)
- [ ] `npx prisma generate` succeeds with the updated schema

## Technical Details
- File: `backend/prisma/schema.prisma`
- The `@default([])` on `String[]` becomes `@default("[]")` on `String`
- All other models (Project, Channel, Message, Task, AgentMemory, ScheduledTask, ScheduledTaskRun, WebhookLog) use only scalar types and `Json` — all SQLite-compatible
- The `@@unique` and `@@index` annotations remain unchanged
