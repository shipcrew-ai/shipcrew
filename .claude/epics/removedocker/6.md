# Task: Update backend code to use JSON helper for agent skills/channels

## Description
Update all 7 backend files that read or write `Agent.skills` and `Agent.channels` to use `parseJsonArray()` on reads and `toJsonString()` on writes. This is the core migration work — mechanical changes at every DB interaction point.

## Acceptance Criteria
- [ ] `backend/src/db/seed.ts` — `JSON.stringify` skills and channels arrays when creating seed agents
- [ ] `backend/src/api/projects.ts` — stringify arrays when creating agents during project provisioning
- [ ] `backend/src/api/agents.ts` — parse skills/channels on read responses, stringify on create/update writes
- [ ] `backend/src/orchestration/router.ts` — parse `agent.channels` before `.includes()` check
- [ ] `backend/src/orchestration/executor.ts` — parse agent skills/channels when reading agent data
- [ ] `backend/src/agents/config.ts` — parse skills/channels when building agent config
- [ ] `backend/src/mcp/tools.ts` — parse agent channels when reading agent data
- [ ] All imports use `import { parseJsonArray, toJsonString } from "../lib/json-fields.js"`

## Technical Details

### Write paths (use `toJsonString`):
- `seed.ts`: Agent creation in seed data — wrap `def.skills` and `def.channels` with `toJsonString()`
- `projects.ts`: Agent creation during project provisioning — same pattern
- `agents.ts`: Agent create/update endpoints — stringify incoming arrays

### Read paths (use `parseJsonArray`):
- `agents.ts`: API responses returning agent data — parse before sending
- `router.ts`: `routeMessage()` checks `a.channels.includes(channel.name)` — parse first
- `executor.ts`: Building execution context from agent record — parse
- `config.ts`: `buildAgentConfigFromDb()` — parse skills/channels
- `tools.ts`: Reading agent channels for MCP context — parse
