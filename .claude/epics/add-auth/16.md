# Task: Create auth middleware and auth API endpoints

## Description
Create the Express auth middleware (`requireAuth`) and the auth router with register, login, and me endpoints.

## Acceptance Criteria
- [ ] `backend/src/auth/middleware.ts` exports `requireAuth` middleware
- [ ] Middleware extracts Bearer token from Authorization header
- [ ] Middleware skips auth for `/health` and paths starting with `/api/auth/`
- [ ] Middleware sets `req.user = { id, email }` on valid token
- [ ] Middleware returns 401 JSON on missing/invalid token
- [ ] TypeScript `Request.user` type augmentation works
- [ ] `backend/src/api/auth.ts` exports auth router with 3 endpoints:
  - POST `/register` — validates with Zod, checks duplicate email (409), hashes password, creates user, returns `{ token, user }`
  - POST `/login` — validates with Zod, finds user, verifies password, returns `{ token, user }` or 401
  - GET `/me` — returns authenticated user info from DB

## Technical Details
- `backend/src/auth/middleware.ts`:
  - Use `declare global { namespace Express { interface Request { user?: ... } } }` for type augmentation
  - Skip list: exact match `/health`, startsWith `/api/auth/`
- `backend/src/api/auth.ts`:
  - Register Zod schema: `{ email: z.string().email(), password: z.string().min(6), name: z.string().min(1) }`
  - Login Zod schema: `{ email: z.string().email(), password: z.string() }`
  - Use prisma.user for DB operations
  - Use hashPassword/verifyPassword from hash.ts
  - Use signToken from jwt.ts

- Task 002 (hash.ts and jwt.ts utilities)
