# Task: Wire auth into server.ts and add Socket.io auth

## Description
Mount the auth middleware and auth router in server.ts. Add Socket.io connection middleware to validate JWT tokens.

## Acceptance Criteria
- [ ] `requireAuth` middleware is applied before all API routers
- [ ] Auth router is mounted at `/api/auth`
- [ ] `/health` endpoint still works without auth
- [ ] `/api/auth/register` and `/api/auth/login` work without auth
- [ ] All other `/api/*` routes require valid Bearer token
- [ ] Socket.io connections require token in `socket.handshake.auth.token`
- [ ] Socket.io rejects connections without valid token

## Technical Details
- File: `backend/src/server.ts`
- Import `requireAuth` from `./auth/middleware.js`
- Import `authRouter` from `./api/auth.js`
- Import `verifyToken` from `./auth/jwt.js`
- Mount order: `app.use(requireAuth)` BEFORE all `app.use("/api/...", ...)` lines
- Mount: `app.use("/api/auth", authRouter)`
- Add `io.use(...)` middleware before `io.on("connection", ...)`:
  ```typescript
  io.use((socket, next) => {
    const token = socket.handshake.auth?.token;
    if (!token) return next(new Error("Authentication required"));
    try {
      const decoded = verifyToken(token);
      socket.data.userId = decoded.userId;
      next();
    } catch {
      next(new Error("Invalid token"));
    }
  });
  ```
